🔬 Firmware Reverse Engineering — Step by Step
1. Acquisition (dump via flash programmer / OTA) 2. Identification (binwalk, file, strings) 3. Extraction (squashfs, cpio, uImage) 4. Static analysis (Ghidra, IDA, radare2) 5. Emulation (QEMU, Firmadyne, Unicorn) 6. Dynamic analysis / debugging

💡 Pro tip: Many IoT devices contain hardcoded SSH keys, API tokens, or backdoor credentials. Always extract the root filesystem and grep for "password", "key", "token", "secret".

🔌 UART / JTAG / SWD — Debug Interface Exploitation
  • → Identifying UART pins (TX, RX, GND, Vcc) with multimeter
  • → Bypassing serial console authentication (bootloader interrupt)
  • → JTAG boundary scan — reading/writing memory directly
  • → SWD (Serial Wire Debug) — dumping MCU flash via OpenOCD
  • → SPI / I2C sniffing for inter-chip communication
